# 软件工程的永生

1. 软件工程客观讲，在人类可枚举的职业中，对整体性和复杂度要求都算得上 TOP10 的，为何它第一时间被 AI 替代了？

2. 早上我让 Antigravity 配置 Gmail MCP Server 增加发送邮件功能，遇到了一个非常有趣的事情，就是安装过程需要 Google 做 Oauth 授权，初始化 Token 遇到了问题，于是 Gemini 3 Pro 整整折腾了一上午（因为中间需要我这个人肉码农点击确认执行），最后把问题解决了，对话记录看的我目瞪口呆，你猜它是怎么把问题解决的？它居然自己下载了源代码，然后分析环境变量的配置，以及对比 Nodejs 和 Python 对 Token 生成的差异，最后自己写了个脚本把 Token 格式转换了，然后就成功了。

3. 这让我对 AI Agent 的理解又有了全新的理解和认识，它工作的方式，像极了我这个老码农，它只是替代了我，而没有替代软件工程本身；

4. 所谓的 SaaS 已死，是软件公司 AI 时代所投入的研发资本，全都成了沉没成本，而软件工程本身不会死，只是旧时王谢堂前燕，飞入寻常百姓家。

5. 回到开篇的问题，为什么要求这么高的职业反而第一时间被替代，就像搞生物分子结构探索研发的两年前被 AlphaFold 替代一样，答案就是‘即时验证’，任务可拆解，且每步骤可以第一时间被验证，这就是 AI 时代的软件工程。

6. 软件工程的本质，就是把复杂问题拆解成可执行的步骤，然后一步步验证，直到解决问题，而 AI Agent 的出现，只是把这个过程自动化了...

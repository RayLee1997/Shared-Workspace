#Meta补票

1. Manus 是工程整合能力上的胜利，最大的亮点并非底层算法能力上创新，而是得益于上游 LLM（特别是 Claude 3.5+）在 Tool Use（Function Calling） 识别精度上的质变。可以说，Manus 是业界最早成功将 LLM 的  CoT 任务规划能力、Google Search API 的上下文补充能力，以及 Python/HTML 代码生成能力深度整合，并在应用层实现多轮任务自动迭代的产品；
    
2. 技术门槛不高，Manus 发布一个月后，我自己就基于开源生态进行了复现验证。仅花费二三周时间，就在 Qwen3上成功跑通了具备类似核心功能的 Deep Research 原型。就目前而言，Manus 早已没了产品功能特性上的秘密和优势，顶级模型本身又进化了；
    
3. Meta 收购 Manus 的核心动力是“补票”，要的是 AI Agent 方面的人。Meta 内部面临的核心矛盾是，前当家掌门人 Yann LeCun 并不完全信奉 Transformer 驱动的 LLM 演进路线。这种技术信仰的分歧导致 Meta 在 AI Model 和 AI Agent 方面掉了大队。同时 Meta 没有自己的 Cloud 业务，导致巨额GPU资本开支只能变成自身的沉默成本，股价和公司士气遭遇打击。这迫使小扎必须支付高昂的溢价进行“补票”，通过引入外部团队来平衡技术路线，确保 Meta 不会在 Agent 时代掉队；
    
4. Manus 年初就提裤跑路破县了，大陆的办公室基本都裁光了，别激动…

# 技术教程梳理工作流

根据用户需求，创建一份结构清晰、实操性强的技术教程。

## 1. 需求确认

首先确认以下信息：

- **教程主题**：用户想要学习的技术/工具是什么？
- **目标受众**：读者的技术背景和经验水平？
- **输出位置**：教程保存到哪个目录？

## 2. 信息收集

// turbo

- 联网搜索该技术的**官方文档**和**最佳实践**
- 查找**配置路径**、**文件格式**、**常用命令**等关键信息
- 收集**实战案例**和**常见问题**

## 3. 教程结构设计

按以下标准模板组织内容：

```markdown
# [技术名称] 完全指南

## 核心概念概览
- 用表格对比核心概念（概念 | 作用范围 | 生命周期 | 典型用途）

## 1. [概念1]
### 怎么操作？
- 配置路径
- 激活方式
### 编写最佳实践
### 示例配置

## 2. [概念2]
（同上结构）

## N. 深度解析：技术实现机制对比
- 用表格对比不同概念的底层机制

## 总结与应用建议
- 学习建议路径
```

## 4. 内容撰写要求

- **明确路径**：所有配置文件必须给出完整、准确的路径
- **区分层级**：区分全局 (Global) 和 工作区 (Workspace) 配置
- **实战案例**：每个概念至少提供一个可直接使用的示例
- **目录结构**：用 `tree` 格式展示项目结构
- **对比表格**：用表格对比相似概念的区别

## 5. 用户反馈迭代

- 根据用户评论反馈，持续优化教程内容
- 补充用户关心的细节（如快捷键、注入机制等）

## 6. 保存输出

将最终教程保存到用户指定的目录，文件名格式：
`[技术名称]_Guide.md` 或 `[技术名称]_完全指南.md`

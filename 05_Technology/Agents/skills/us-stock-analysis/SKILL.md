# US Stock Analysis（美股个股深度分析）

## 功能概述

通过整合 **Brave Search MCP**、**EdgarTools MCP** 和 **Yahoo Finance MCP** 三大数据源，对美股上市公司进行**长篇深度分析**：

1. **用户意图拆解**：系统性分解用户需求，明确分析维度和深度
2. **SEC 官方数据**：10-K/10-Q/8-K 财务报表、重大事件披露
3. **电话会议纪要**：提取过去4个季度的 Earnings Call 要点，**重点解读 CEO Q&A 环节**
4. **季度数据追踪**：以表格形式呈现过去1年每季度关键业务指标
5. **深度业务洞察**：基本面分析、竞争格局、战略方向、风险评估
6. **市场行情与持仓**：通过 Yahoo Finance MCP 获取实时股价、历史走势、机构持仓、分析师评级等市场侧数据

**核心输出**：一份**长篇、结构化、数据驱动**的深度研究报告

---

## 使用时机

当用户需要以下内容时使用此技能：

- **个股研究**：「帮我分析一下 AAPL 的财务状况」「深度分析 NVDA」
- **财报解读**：「解读 Tesla 2025年Q4财报」「PLTR 最近的财报电话会议说了什么」
- **公司尽调**：「我想了解 Palantir 的商业模式和财务健康」
- **投资决策支持**：「分析一下 Meta 是否值得投资」
- **行业对比**：「对比软件行业几家头部公司的财务表现」

**不要使用**：

- 非美股公司（如 A股、港股）
- 纯技术面/K线分析（本技能专注基本面）
- 已有充分信息的简单问答

## 执行流程（必须遵循）

### 第 1 步：用户意图深度拆解

**这是核心步骤**，必须系统性分解用户需求：

#### 1.1 识别分析对象

```
分析对象拆解：
├── 公司标识：Ticker（如 PLTR）/ 公司全称 / 别名
├── 所属行业：软件、半导体、银行、制药等
├── 公司阶段：成长期 / 成熟期 / 转型期
└── 特殊属性：FAANG / 中概股 / SPAC / 近期IPO
```

#### 1.2 确定分析类型与深度

| 分析类型 | 用户意图关键词 | 分析深度 | 必须包含的模块 |
|----------|---------------|----------|---------------|
| **全面深度分析** | 「深度分析」「全面了解」「研究一下」 | ★★★★★ | 全部模块 |
| **财报解读** | 「财报」「季报」「年报」「业绩」 | ★★★★ | 财务数据 + 电话会议 |
| **业务分析** | 「商业模式」「竞争力」「护城河」 | ★★★★ | 业务基本面 + 竞争格局 |
| **管理层追踪** | 「CEO」「管理层」「战略」 | ★★★ | 电话会议 Q&A + 新闻 |
| **快速概览** | 「简单看看」「大概情况」 | ★★ | 核心指标 + 摘要 |

#### 1.3 明确时间范围

> ⚠️ **财年 ≠ 日历年**：很多公司财年结束月份不是12月。例如 某公司的财年结束于2月，其 FY2026 = 2025年3月 - 2026年2月。**分析前务必先确认目标公司的财年定义。**

| 范围类型 | 示例 | 数据量 |
|----------|------|--------|
| **默认** | 最新财报 | 过去 4 季度 |
| **用户指定** | 「最近两年」 | 8 季度 |
| **特定季度** | 「Q4 2025」 | 单季度深度 |

#### 1.4 生成分析计划

在开始数据收集前，**必须先输出分析计划**：

```markdown
## 📋 分析计划

**分析对象**：[公司名称] ([Ticker])
**分析类型**：[全面深度分析 / 财报解读 / 业务分析 / ...]
**时间范围**：[如：2025 Q1-Q4，共4个季度]
**特别关注**：[用户特别提及的方面]

**数据收集清单**：
- [ ] SEC 财务数据（4个季度）
- [ ] 公司概况与近期文件
- [ ] Q1 2025 Earnings Call Transcript
- [ ] Q2 2025 Earnings Call Transcript
- [ ] Q3 2025 Earnings Call Transcript
- [ ] Q4 2025 Earnings Call Transcript
- [ ] 重大新闻事件
- [ ] 行业竞争格局

**预计输出**：长篇深度分析报告（约 5000-8000 字）
```

### 第 2 步：获取 SEC 官方数据（EdgarTools）

**优先调用 EdgarTools**，这是最权威的官方数据来源。

```
# 1. 公司概览 + 近期文件
edgar_company_research:
  identifier: "PLTR"
  detail_level: "detailed"
  include_filings: true
  include_financials: true

# 2. 过去4个季度财务数据（关键！用于季度数据表格）
edgar_analyze_financials:
  company: "PLTR"
  statement_types: ["income", "balance", "cash_flow"]
  periods: 4                   # 4个季度
  annual: false                # 季度数据

# 3. 过去3年年度数据（用于长期趋势）
edgar_analyze_financials:
  company: "PLTR"
  statement_types: ["income", "balance", "cash_flow"]
  periods: 3
  annual: true                 # 年度数据

# 4. 行业对比（可选但推荐）
edgar_industry_overview:
  industry: "software"
  include_top_companies: true
  limit: 10
```

> 💡 **营收结构拆解**：从上述 income statement 数据中提取 Revenue by Segment、Cost of Revenue、Gross Profit、OpEx (R&D + SG&A)、Operating Income、Tax & Interest、Net Income，用于填充报告头部的「💰 最新季度财报营收结构拆解」章节。

### 第 3 步：检索电话会议纪要（重点步骤）

**必须获取过去4个季度的 Earnings Call Transcript**，这是理解管理层思路的核心来源。

#### 3.1 电话会议搜索策略

```
# 逐季度搜索（必须并行执行以提高效率）
brave_web_search:
  query: "[公司名] Q4 2025 earnings call transcript full"
  count: 10

brave_web_search:
  query: "[公司名] Q3 2025 earnings call transcript"
  count: 10

brave_web_search:
  query: "[公司名] Q2 2025 earnings call transcript"
  count: 10

brave_web_search:
  query: "[公司名] Q1 2025 earnings call transcript"
  count: 10
```

#### 3.2 优质 Transcript 来源优先级

1. **官方 Investor Relations 页面**（最权威）
2. **Seeking Alpha**（较完整，含 Q&A）
3. **The Motley Fool**（结构清晰）
4. **Yahoo Finance**（快速但可能不完整）
5. **其他财经媒体**

#### 3.3 CEO Q&A 环节提取要点

**必须重点关注 CEO 在 Q&A 环节的回答**：

| 关注维度 | 提取要点 |
|----------|----------|
| **业务展望** | 对下季度/全年的指引（Guidance） |
| **战略方向** | 新产品、新市场、并购计划 |
| **竞争态势** | 对竞争对手的评价、市场份额变化 |
| **风险披露** | 管理层主动提及的挑战和风险 |
| **客户动态** | 大客户签约、流失、续约情况 |
| **技术路线** | AI、云、新技术的投入和进展 |
| **资本配置** | 回购、分红、投资计划 |

### 第 4 步：检索行业竞争对手数据

**必须识别并分析 3-5 家主要竞争对手**，获取他们的最新财务数据和战略投入。

#### 4.1 识别主要竞争对手

```
# 方法1：通过 EdgarTools 行业概览
edgar_industry_overview:
  industry: "[行业类型]"  # 如：software, semiconductors, banking
  include_top_companies: true
  limit: 10

# 方法2：通过搜索识别
brave_web_search:
  query: "[目标公司] main competitors [行业]"
  count: 10
```

#### 4.2 获取竞争对手财务数据

**对每个主要竞争对手，必须获取**：

```
# 对每个竞争对手执行（以竞争对手1为例）
edgar_company_research:
  identifier: "[竞争对手 Ticker]"
  detail_level: "standard"
  include_financials: true

edgar_analyze_financials:
  company: "[竞争对手 Ticker]"
  statement_types: ["income", "balance", "cash_flow"]
  periods: 4                   # 最新4个季度
  annual: false
```

#### 4.3 检索竞争对手战略动态

**重点关注竞争对手的战略投入和业务方向**：

```
# 财报电话会议（获取战略重点）
brave_web_search:
  query: "[竞争对手名] latest earnings call transcript [最新季度]"
  count: 5

# 战略投资和产品动态
brave_web_search:
  query: "[竞争对手名] AI investment strategy 2025"
  count: 5

brave_web_search:
  query: "[竞争对手名] new product launch 2025"
  count: 5

# 资本开支和研发投入
brave_web_search:
  query: "[竞争对手名] capex R&D spending [最新季度]"
  count: 5
```

#### 4.4 竞争对手数据提取要点

**必须提取以下信息**（每个竞争对手）：

| 数据类型 | 具体指标 |
|----------|----------|
| **财务表现** | 最新季度营收、同比增长、毛利率、净利率、EPS |
| **业务指标** | 客户数、ARR/MRR、净收入留存率（如适用） |
| **战略投入** | AI/新技术投资金额、研发占比、资本开支重点 |
| **市场份额** | 在目标市场的份额、增长趋势 |
| **管理层重点** | 电话会议中强调的战略优先级 |
| **竞争优势** | 独特的产品/技术/市场地位 |

### 第 6 步：信息整合与迭代

**信息不足时的迭代策略**：

| 信息缺口 | 迭代策略 |
|----------|----------|
| 某季度 Transcript 找不到 | 尝试：「[公司名] [季度] [年份] earnings call」→「[CEO姓名] [季度] remarks」 |
| 财务数据不完整 | 扩展 EdgarTools periods 参数至 8 个季度 |
| CEO Q&A 内容不足 | 搜索「[CEO姓名] investor day」「[CEO姓名] conference presentation」 |
| 行业背景不清 | 调用 `edgar_industry_overview` + 搜索行业报告 |

**迭代终止条件**：

- 已获取 ≥3 个季度的 Earnings Call 关键信息
- 已获取 4 个季度的财务数据
- 或迭代 ≥3 轮仍无新增有效信息（应在报告中说明数据限制）

### 第 7 步：深度分析与洞察

#### 7.1 财务健康评估框架

| 维度 | 关键指标 | 健康信号 | 警示信号 |
|------|----------|----------|----------|
| **成长性** | 营收增长率、客户增长 | >20% YoY | <0% 或连续下滑 |
| **盈利能力** | 毛利率、净利率、EBITDA Margin | 稳定或改善 | 持续下滑 |
| **现金流** | 经营现金流、FCF | 正且增长 | 持续为负 |
| **资产负债** | 负债率、流动比率 | <50%, >1.5 | >80%, <1.0 |
| **效率指标** | ROE, ROIC | >15% | <5% |

#### 7.2 业务基本面分析框架

```
业务基本面深度分析：
├── 商业模式
│   ├── 价值主张：公司解决什么问题？
│   ├── 收入模式：如何赚钱？（订阅/授权/交易/广告）
│   ├── 客户结构：B2B/B2C？大客户依赖度？
│   └── 定价能力：是否有提价空间？
│
├── 竞争优势（护城河）
│   ├── 网络效应：用户越多价值越大？
│   ├── 转换成本：客户迁移成本高吗？
│   ├── 品牌价值：品牌溢价能力？
│   ├── 成本优势：规模效应/独特资源？
│   └── 技术壁垒：专利/独特技术？
│
├── 增长驱动力
│   ├── 市场扩张：TAM（总可寻址市场）增长
│   ├── 产品创新：新产品/功能推出
│   ├── 客户扩展：新客户获取 + 老客户增购
│   └── 并购整合：战略收购带来的协同
│
└── 风险因素
    ├── 竞争风险：新进入者、替代品
    ├── 监管风险：政策变化、合规成本
    ├── 技术风险：技术迭代、被颠覆
    ├── 客户风险：大客户流失、集中度
    └── 宏观风险：经济周期、利率、汇率
```

#### 7.3 电话会议要点提炼

**每个季度必须提炼以下内容**：

1. **管理层开场陈述要点**（3-5 点）
2. **关键业务指标更新**
3. **CEO 在 Q&A 中的核心回答**（至少 3 个有价值的问答）
4. **前瞻性指引（Guidance）**
5. **管理层语气/信心变化**

### 一句话结论
[用一句话概括投资结论]

### 核心发现
| 维度 | 评估 | 要点 |
|------|------|------|

### 命名格式

**标准格式**：`{Ticker大写} 深度分析报告.md`

- **公司名称**：使用英文全称或常用简称（如 Amazon、Palantir、Apple）
- **美股代码**：大写的股票代码（如 AMZN、PLTR、AAPL）
- **财报季度代码**：使用以下格式之一
  - `Q{季度}_{财年}` - 例如：`Q4_FY2025`（财年第4季度）、`Q1_FY2026`（财年第1季度）
  - `Q{季度}_{日历年}` - 例如：`Q4_2025`（2025年第4季度）、`Q1_2026`（2026年第1季度）

⚠️ **重要提示**：

- 财年（FY）≠ 日历年：很多公司财年与日历年不一致，务必根据公司实际财年命名
- 优先使用财年格式（FY），除非公司财年与日历年一致
- 季度代码应使用报告中分析的**最新季度**

## 错误处理

| 情况 | 处理方式 |
|------|----------|
| EdgarTools 无数据 | 检查 Ticker 是否正确，尝试用公司名称 |
| 某季度 Transcript 找不到 | 在报告中注明「该季度电话会议纪要未能获取」，继续其他部分 |

| 财务数据冲突 | 以 SEC 官方文件为准，标注差异 |
| CEO Q&A 内容不完整 | 尽量补充 Investor Day、Conference 等其他来源 |
| Yahoo Finance 网络超时 | Yahoo Finance 偶有访问限制，稍后重试；财务数据以 EdgarTools 为准 |
| yfinance 返回空数据 | 确认 Ticker 正确且为美股代码；某些小市值公司数据可能不全 |
